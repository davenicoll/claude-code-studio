## 自動承認ポリシー

以下のコマンドは自動的に承認されます：

- ファイルの読み取り・一覧表示
- ビルド・コンパイル (`npm run build`, `tsc`, etc.)
- テスト実行 (`npm test`, etc.)
- 開発サーバー起動 (`npm run dev`, etc.)
- パッケージインストール (`npm install`, etc.)
- コード生成・フォーマット (`prettier`, `eslint --fix`, etc.)
- Git操作（コミット、ブランチ作成、プッシュなど通常操作）
- ローカルファイルの作成・編集

## 自動承認しない操作（必ずユーザー確認を求める）

- 外部APIの呼び出し（課金が発生する可能性があるもの）
- 情報の外部公開（公開設定の変更など）
- 不要なファイル・ディレクトリの削除
- GitHubリポジトリの破壊的操作（force push、ブランチ削除、リポジトリ削除など）
